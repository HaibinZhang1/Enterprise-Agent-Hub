# 19. 视觉设计系统与页面统一规范

## 页面目标
为新增页面、模块和状态视图提供统一的设计语言、布局规则、组件规格与动效约束，使 AI 或前端实现时可以直接参照本规范，保持页面风格一致、密度一致、交互节奏一致。

> 说明：本规范是当前版本 Desktop 工作台的页面视觉基线。其设计来源综合自 `AuraForge`、`Material Redesign`、`Layout Prototype` 与 `Desktop Design Prototype` 四组原型。新增页面默认采用“浅色企业 AI 工作台”风格；深色方案仅用于管理、审计、沉浸式工作区等少数场景，不作为默认页面底色。

---

## 19.1 设计定位

### 19.1.1 统一风格定义
当前产品的统一视觉方向定义为：

**浅色企业 AI 工作台 + 克制微动效 + 桌面级高密度操作区**

补充品牌口径：当前界面应表达“科技、快速、工具”的气质。可参考 Raycast 的即时工具感与 Linear 的清晰层级，但不得走向繁杂、炫技或营销化。

### 19.1.2 风格关键词
- 专业、可信、快速、工具化
- AI 原生，但不做科幻炫技
- 信息密度高，但层级必须清楚
- 默认桌面端优先，支持窄宽度收缩
- 数据、状态、目标、结果优先于装饰性内容
- 视觉记忆点来自干净的命令工作台、稳定的 Inspector 和清晰状态，而不是装饰图形

### 19.1.3 新页面必须满足的感知结果
- 第一眼能看出是同一产品，而不是另一套站点
- 用户在 3 秒内能定位页面主任务、主操作、主状态
- 卡片、按钮、标签、输入框在不同页面上保持同一语法
- 页面即使模块变多，也不能退化成“堆卡片”

### 19.1.4 明确禁止
- 随机换主题色、随机换圆角体系、随机换阴影体系
- 为了“更像 AI”加入大量光效、粒子、发光描边、过度渐变
- 页面级营销 Hero 抢占工作区，影响操作密度
- 单页同时出现多种不兼容风格，例如企业后台 + 霓虹赛博 + Material 夸张色块
- 用插画替代系统状态、数据和结构表达

## 19.2 主题策略

### 19.2.1 默认主题
- 默认使用浅色主题
- 页面底色保持冷白、浅灰、雾面蓝灰体系
- 强调“清洁桌面工作区”，而不是“深色代码编辑器”
- 当前主题入口仅提供经典白与暗色；不得展示尚未实现的清爽蓝、高对比等主题选项

### 19.2.2 默认浅色主题中的深色区块边界
仅以下场景允许采用深色主背景：
- 管理/审计类高密度工作台
- 需要突出矩阵、Inspector、风险或系统约束的页面
- 全屏沉浸式流程编排画布
- 预览区、终端区、日志区、差异对比区

### 19.2.3 应用内手动暗色主题
- 设置中的主题选项允许用户手动选择应用级全局暗色主题
- 暗色主题为软件层面的持久化偏好，不自动跟随操作系统主题
- 暗色主题必须通过共享 Token 覆盖全局画布、卡片、浮层、输入、边框、文字、状态色与反馈提示，避免只在局部组件追加一次性深色样式
- 默认主题仍为浅色，浅色主题中的深色区块边界继续遵循 19.2.2

### 19.2.4 同页混用规则
- 同一页面允许有 1 个深色强调区块，例如 featured banner、日志面板或沉浸工作区
- 除该强调区块外，其余卡片仍保持浅色体系
- 不允许整页出现多个互相竞争的深色强调区

### 19.3.3 渐变规则
- 默认页面、按钮、品牌徽标、头像、Skill 首字母图标和详情标识不得使用高饱和蓝紫渐变。
- 如确需渐变，只允许使用灰绿深青的低透明度环境层，并且只能作为 Hero 或 Featured Banner 的弱背景，不作为主操作或列表图标底色。
- 不允许整页铺满大面积高饱和渐变底。

| `pine` | `#315F5A` | `#DCE9E4` | 品牌、登录、主对象 |
| `eucalyptus` | `#4D6D63` | `#E1EEE9` | 治理、安全、自动化 |
| `seafoam` | `#3F6F6B` | `#DFF0ED` | 更新、同步、连接 |
| `steel` | `#566877` | `#E5EBEE` | 工具、项目、文件 |
| `mauve-gray` | `#6C626D` | `#ECE8ED` | 审核、工作流、关系对象 |

## 19.4 字体与排版

### 19.4.1 字体族
- UI 正文字体：系统原生 UI 字体栈，Windows 优先 `Segoe UI Variable Text`，中文优先 `PingFang SC` / `Microsoft YaHei UI`
- 展示型标题字体：同源系统展示字体栈，Windows 优先 `Segoe UI Variable Display`
- 数值、版本、路径、Hash、状态码：等宽字体栈 `ui-monospace` / `SF Mono` / `Menlo` / `Consolas`
- 不新增远程 Web Font 依赖；桌面客户端优先启动速度、中文渲染稳定性和系统一致性

### 19.4.2 排版等级

| 等级 | 字号/行高 | 字重 | 用途 |
|------|-----------|------|------|
| Display XL | `40 / 48` | 800 | 页面主 Hero，仅首页或核心工作台允许 |
| H1 | `32 / 40` | 800 | 页面主标题 |
| H2 | `24 / 32` | 700 | 区块标题 |
| H3 | `20 / 28` | 700 | 卡片组标题、详情模块标题 |
| H4 | `16 / 24` | 700 | 卡片标题、行标题 |
| Body M | `14 / 22` | 500 | 标准正文 |
| Body S | `13 / 20` | 500 | 次级正文 |
| Meta | `12 / 18` | 500 | 描述、注释、辅助说明 |
| Label | `11 / 16` | 700 | Section Label、Badge、统计标签 |
| Micro | `10 / 14` | 700 | 状态小标、角标、按钮次级说明 |

描述性文字规则：主要描述使用 `Body M`、`text-secondary`；字段提示、元信息和辅助说明使用 `Meta`、`text-muted`；禁用态或极弱提示才使用 `text-soft`。列表项描述尽量控制在 1 到 2 行，避免用高饱和强调色承载正文说明。

### 19.4.3 文案约束
- 中文产品文案优先简洁、短句、结果导向
- 页面标题描述主任务，不描述页面本身
- 列表项描述尽量控制在 1 到 2 行
- Meta 文案不堆砌名词，优先“对象 + 状态 + 动作”

### 19.4.4 标题使用规则
- 展示字体 token 仅用于页面主标题、Hero、指标大数值、少量展示型模块
- 常规后台列表、表格、详情说明统一使用 UI 字体 token
- 一个页面最多 1 个 Display 级标题，避免争抢视觉中心
- 字距保持 `0`；不得为了“高级感”使用负字距

## 19.5 间距、圆角与阴影

### 19.5.1 间距体系
采用 4pt 基础间距体系，常用语义 token 为 `--space-2xs / xs / sm / md / lg / xl / 2xl`。允许值：
- `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96`

### 19.5.2 典型间距

| 场景 | 取值 |
|------|------|
| 页面左右边距 | 桌面工作台默认 `20`，窄屏 `12` |
| 顶栏高度 | 当前 Desktop 工作台实现基准为 `60`，窄屏允许换行但不得横向溢出 |
| 侧栏宽度 | 社区、本地、管理统一为 `220` |
| Inspector / 热榜栏宽度 | 桌面统一为 `340` |
| 页面主区块间距 | `16` |
| 卡片内边距 | 标准业务面板 `18`，列表行 `12` 到 `14` |
| 大型 Hero / Featured 面板内边距 | `32` 到 `40` |
| 图标与标题间距 | `10` 到 `12` |
| 按钮组间距 | `8` 到 `12` |

### 19.5.3 圆角梯度

| 名称 | 值 | 用途 |
|------|------|------|
| `radius-xs` | `6` | 小徽标、小标签 |
| `radius-sm` | `8` | 小按钮、小输入框、小卡片 |
| `radius-md` | `12` | 输入框、列表行、标准控件 |
| `radius-lg` | `16` | 节点卡、模块块 |
| `radius-xl` | `20` | 常规业务卡片 |
| `radius-panel` | `14` | 标准工作台面板、Inspector、空状态、弹窗外框 |
| `row-radius` | `12` | 列表行、市场行、排行榜行 |
| `radius-pill` | `999` | Chip、Badge、计数胶囊 |

按钮圆角规则：所有按钮统一使用 `radius-sm = 8`；顶部一级导航分段按钮内部 active 项使用 `radius-xs = 6`。按钮不得临时使用 `999` 胶囊圆角、`16` 或更大的卡片圆角；大按钮只能通过高度、宽度、字号表达强调。

### 19.5.4 阴影层级

| 层级 | 值 | 用途 |
|------|------|------|
| `shadow-xs` | `0 1px 2px rgba(15,23,42,0.04)` | 顶栏、轻面板、品牌徽标 |
| `shadow-sm` | `0 8px 20px rgba(15,23,42,0.05)` | 命令输入框、少量强调面 |
| `shadow-lg` | `0 22px 56px rgba(15,23,42,0.12)` | Popover、弹窗、Toast |

### 19.5.5 边框规则
- 优先细边框，而不是重阴影
- 标准边框透明度控制在 `6%` 到 `12%`
- 选中态以“边框提亮 + 轻微阴影抬升”为主，不做粗边框硬切换

## 19.6 页面骨架与布局模板

### 19.6.1 全局骨架
新增页面必须优先落入以下统一骨架：

```text
顶栏（60）
├─ 左：品牌/返回/面包屑
├─ 中：全局搜索或页签导航
└─ 右：通知/筛选/主操作/账号

主体区
├─ 左侧一级导航（当前 Desktop 工作台为 220，可收起）
└─ 主内容区（工作台栏间距 16，整页内容按 24~32 内边距）
```

### 19.6.1.1 当前 Desktop 工作台布局基准
当前生产桌面端以 **本地 > Skills** 页面作为工作台尺寸基准。实现层必须复用 `apps/desktop/src/styles.css` 中的布局 token，不再为单个页面临时硬编码栏宽：

| Token | 值 | 用途 |
|------|------|------|
| `--layout-gap` | `16px` | 顶栏与主体、侧栏与内容栏、列表栏与简介栏之间的标准间距 |
| `--layout-rail-width` | `220px` | 社区、本地、管理等工作台左侧导航栏宽度 |
| `--layout-inspector-width` | `340px` | 右侧简介栏、Inspector 或治理面板宽度 |
| `--layout-topbar-min-height` | `60px` | 顶部导航栏最小高度 |
| `--layout-panel-padding` | `18px` | 标准业务面板默认内边距 |

标准工作台四区如下：

```text
顶部导航栏：最小高度 60px
主体区：左侧导航栏 220px + 16px 间距 + 主内容区
主内容区常规双栏：列表/主内容栏 minmax(0, 1fr) + 16px 间距 + 简介/Inspector 栏 340px
```

允许的布局例外：
- 社区 Skill 页采用单栏结果区，列表栏内部可以拆为两列卡片网格；榜单仅在社区首页展示。
- 发布流程、首页、详情全屏阅读等场景可以将列表栏与简介栏合并为整页内容，但仍要复用顶栏高度、页面间距和面板滚动边界。
- 管理部门页可以保留三列：部门树 / 中心工作区 / Inspector，其中右侧 Inspector 必须保持 `340px`，部门树与中心工作区使用统一 `16px` 间距。

### 19.6.2 主内容区的四种标准模板

#### A. 概览页
适用于：首页、总览、系统入口
- 顶部标题区
- 1 个主 Hero 或系统摘要卡
- 2 到 4 个指标卡
- 1 到 2 个主任务模块
- 1 个次级活动流/日志区

#### B. 工作台页
适用于：已安装、工具与项目、编排
- 左列：对象列表
- 中列：矩阵/主操作区
- 右列：Inspector / 结果 / 异常说明
- 三列需要在同屏内互相联动

#### C. 市场页
适用于：市场、插件、模板
- 标题区
- 筛选/搜索/排序条
- 1 个精选模块（可选）
- 卡片网格区
- 列表底部支持空状态与分页/延迟加载

#### D. 管理页
适用于：审核、部门、用户、Skill 管理
- 左侧二级导航或分组入口
- 中列列表
- 右列详情或操作面板
- 风格允许更深、更密，但结构不变

### 19.6.3 工作台滚动边界
社区、本地、管理等桌面工作台页默认不再在主内容顶部放独立标题栏。主任务、筛选、搜索、排序与操作按钮应放在对应业务卡片的顶部。

列表类卡片必须保持固定边界：
- 页面顶栏、左侧二级导航与业务卡片外框不跟随列表滚动。
- 搜索、筛选、分段按钮、主操作按钮固定在列表卡片顶部。
- 滚动只发生在卡片内部的列表、网格、树或详情内容区域。
- 管理用户、部门树、审核单、Skill 网格、本地工具/项目等长列表必须遵守同一滚动规则。

布局验收规则：
- 桌面断点下，社区、本地、管理工作台的左侧导航栏宽度必须一致，为 `220px`。
- 双栏工作台的右侧简介/Inspector 栏必须一致，为 `340px`，允许 `±2px` 浏览器渲染误差。
- 侧栏、列表栏、简介栏之间的水平间距必须一致，为 `16px`。
- 列表栏和简介栏外框顶部、底部必须在同一工作区内对齐；长内容只能在栏内部滚动。
- 窄屏按现有响应式规则折叠：`1200px` 以下优先单列，`800px` 以下侧栏可横向排列且内容不得溢出。

### 19.6.4 页面新增时的模板选择原则
- 以“主任务”而不是“页面名”决定模板
- 不能确定时优先选 `B. 工作台页`
- 只有真正需要“发现与比较”的场景才用 `C. 市场页`

## 19.7 背景、装饰与视觉氛围

### 19.7.1 页面背景
- 主背景默认使用 `--canvas`
- 可叠加非常克制的冷白到雾面蓝灰环境渐变
- 不使用装饰性网格、粒子、光斑或与业务状态无关的背景图形
- 背景不能抢占工作区信息密度；列表、Inspector 和操作区必须保持最高可读性

### 19.7.2 环境光
- 允许的环境层仅限 `--body-background`、`--surface-panel-gradient`、`--surface-command-gradient`
- 不新增独立光斑、装饰球、bokeh 或大面积发光块
- 不允许出现高饱和霓虹光晕或蓝紫渐变

### 19.7.3 玻璃效果
- `backdrop-filter: blur(...)` 只用于：
  - 顶栏
  - 浮层 / 菜单 / 弹窗
- 普通业务卡片仍以实体白卡为主

### 19.7.4 渐变与插图
- 渐变是点缀，不是底色主角
- 页面若有一处深色 Banner，则其他区域必须回归克制
- 不允许用插图替代结构布局

---

## 19.8 图标系统

### 19.8.1 图标来源
- 默认使用 `Lucide` 风格线性图标
- 品牌型对象可使用品牌色容器，但图形仍保持简洁

### 19.8.2 图标尺寸

| 场景 | 尺寸 |
|------|------|
| 输入框、按钮内图标 | `16` |
| 顶栏、导航、行操作 | `18` |
| 卡片头部主图标 | `20` 到 `24` |
| 空状态/上传区图标 | `32` 到 `40` |
| Hero / Featured 视觉图标 | `48` 到 `80` |

### 19.8.3 图标容器规格

| 场景 | 容器尺寸 | 圆角 |
|------|-----------|------|
| 小按钮内图标容器 | `28` 到 `32` | `8` |
| 标准卡片图标容器 | `40` 到 `48` | `8` |
| 大卡片图标容器 | `56` 到 `64` | `8` |

### 19.8.4 图标颜色规则
- 默认图标容器使用低饱和浅底 + 单色前景，不使用高饱和蓝色或蓝紫渐变
- Skill、部门、用户、项目等列表图标可使用不同 tone，但必须由实体 ID 或名称稳定映射
- 每张卡片最多 1 个图标容器；Lucide 线性图标继承父级 `currentColor`
- Marketplace / Plugin 卡如需品牌识别，也必须降低饱和度，不能压过列表正文
- 状态图标优先用 dot、check、alert、loader，不滥用装饰图标

### 19.8.5 禁止事项
- 不混用线性图标和拟物图标
- 不把所有默认图标统一染成高饱和蓝色
- 不在列表中使用蓝紫渐变色块作为默认图标背景
- 不使用厚重 3D icon pack
- 不为普通业务卡片添加无意义大插画

## 19.9 卡片系统

### 19.9.1 指标卡
- 圆角：`8` 到 `14`
- 内边距：`14` 到 `18`
- 结构：Label / Value / Secondary Hint
- 数值推荐使用等宽字体
- 适用于：总数、延迟、容量、活跃状态

### 19.9.2 实体卡
- 圆角：优先 `row-radius = 12`，真正独立的大块才使用 `radius-panel = 14`
- 结构：图标区 / 标题描述 / 状态 / Footer Meta
- hover 时只做轻微底色或边框变化，不做明显位移
- 适用于：Skill、插件、Agent、模板、部门、工具

### 19.9.3 列表行卡
- 行高：`64` 到 `84`
- 圆角：`12`
- hover 只做底色变化或轻微提亮
- 列表行 hover 不做明显位移，优先使用主色 `3%` 到 `6%` 的浅染色背景
- 适用于：日志、通知、上下文注入记录、审核列表

### 19.9.4 上传/空状态卡
- 圆角：`14`
- 虚线边框：`1px`
- 图标置中
- 说明文案控制在 2 行内
- 适用于：拖拽上传、空列表、空结果

### 19.9.5 Inspector 卡
- 圆角：`14`
- 分块展示“当前选中 / 最终结果 / 异常说明 / 主动作”
- 底部主动作固定在视觉闭环位置
- 适用于：部署、审核详情、工具冲突、权限说明

### 19.9.6 Featured Banner
- 每页最多 1 个
- 圆角：`16` 到 `20`
- 内边距：`32` 到 `40`
- 可以采用 `--surface-command-gradient` 或低饱和深色强调面
- 右侧允许出现大图标或抽象图形
- 不承载复杂表单与多行操作

## 19.10 控件与组件规格

### 19.10.1 按钮

| 类型 | 高度 | 圆角 | 说明 |
|------|------|------|------|
| Primary | `40` | `8` | 灰绿深青实色底，承载主操作 |
| Secondary | `40` | `8` | 白底边框，承载次操作 |
| Filled Tonal | `40` | `8` | `primary-container` 浅底，用于次重点操作 |
| Large Primary | `44` 到 `64` | `8` | Hero、上传、发布等大按钮 |
| Icon Button | `36` 到 `40` | `8` | 过滤、切换、设置 |

规则：
- 一个操作区只允许 1 个明确主按钮
- 同一行按钮不超过 3 个
- 主按钮文案优先动词开头，如“安装”“发布 Skill”“创建流程”
- 除顶部一级导航分段按钮外，所有按钮圆角统一为 `8`
- Primary 不使用渐变；hover 只略微加深为 `primary-hover`
- Secondary / Ghost / Tonal / Icon 按钮 hover 统一使用 `state-layer`
- Focus 使用 `--focus-shadow`，disabled 统一降低透明度、移除阴影和位移

### 19.10.2 输入框
- 高度：`40` 到 `44`
- 圆角：`8`
- 左侧允许搜索图标
- focus ring 统一为主色 `4px` 轻透明外发光
- placeholder 使用 `text-soft`

### 19.10.3 分段导航 / Tabs / Chips
- 高度：`32` 到 `40`
- 一级导航外框圆角 `8`，内部 active 项圆角 `6`
- Chip / Badge / 计数胶囊圆角 `999`
- 选中态优先深色底或实体浅色底
- 未选中态保持弱背景 + 细边框
- hover 时允许轻微变色，推荐使用主色 `4%` 到 `6%` 的浅染色背景，不做明显跳变
- 同一分组内标签数建议不超过 8 个

### 19.10.4 Badge / 状态标签
- 高度：`20` 到 `24`
- 字号：`10` 到 `11`
- 字重：700
- 推荐大写或短词，不超过 14 个字
- 同一卡片状态标签不超过 2 个

### 19.10.5 Progress / Metric 条
- 进度条高 `6`
- 轨道使用浅灰
- 进度使用主渐变或单色
- 数值说明置于条上方右侧，不放在条中间

## 19.11 状态、反馈与空错场景

### 19.11.1 必须统一处理的状态
- 默认
- hover
- active
- selected
- disabled
- loading
- success
- warning
- danger
- empty
- offline

### 19.11.2 状态表现原则
- 成功：绿色标签或绿色 dot，避免大面积绿色底
- 风险：橙色或黄色信息条，必须附带原因
- 错误：红色仅用于真实失败、危险或不可恢复动作
- 加载：优先 spinner、skeleton、局部状态替换，不要整页闪烁
- hover：在原有轻微浮起或提亮基础上，必须补充轻微变色；浅色主题默认使用主色 `3%` 到 `8%` 的染色背景，避免只靠阴影表达可点击性

### 19.11.3 空状态文案结构
- 第一行：说明“当前没有什么”
- 第二行：说明“为什么”或“下一步能做什么”
- 允许带 1 个主操作按钮

---

## 19.12 动效规范

### 19.12.1 动效原则
- 动效服务于状态切换、可点击感和层级变化
- 不服务于单纯炫技
- 动效不能干扰阅读和操作效率

### 19.12.2 时间曲线

| 场景 | 时长 | 曲线 |
|------|------|------|
| Hover / active | `--motion-fast = 160ms` | `ease` |
| 控件切换 | `--motion-base = 180ms` | `ease` |
| 弹窗 / Toast 出入场 | `180ms` 到 `240ms` | `ease` |
| 页面切换 | `320` 到 `400ms` | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 侧栏开合 | `300ms` | `ease-in-out` |
| 骨架/脉冲 | `1.6` 到 `2s` | `ease-in-out` |
| Spinner | `1s` | `linear infinite` |

### 19.12.3 允许的动效形式
- `opacity`
- `translateY(-1 ~ -4px)`
- `scale(1.00 ~ 1.01)`
- `border-color`
- `shadow`
- `background-color`
- 状态点 pulse
- loader spin
- 轻量 fade up

### 19.12.4 标准动效模板
- 页面视图切换：`fadeUp(8px -> 0, 320ms)`，仅用于真正的页面切换
- 卡片 / 列表 hover：默认不位移，只叠加主色 `4%` 到 `8%` 的轻微染色背景与边框变化
- 侧栏开关：`translateX`
- 状态同步：旋转 loader
- 在线状态：小圆点 `pulse`
- 长循环动效仅用于真实运行状态，不用于装饰

### 19.12.5 明确禁止
- 大幅缩放
- 弹簧式弹跳
- 高频闪烁
- 持续漂浮、持续抖动
- 页面级自动轮播和大面积自动位移

### 19.12.6 减弱动效要求
- 需要支持减少动效模式
- 降级后仅保留 opacity、颜色与状态切换
- 长循环动画应能关闭

## 19.13 页面级组件清单

新增页面默认只能从以下组件集合中组合，不建议临时发明新的大类组件：

1. 顶栏 Header
2. 一级侧栏 Navigation Rail
3. 二级分组导航 / Segment Tabs
4. 页面标题区
5. 指标卡
6. 实体卡
7. 列表行
8. Featured Banner
9. 空状态/上传卡
10. Inspector 面板
11. 筛选栏 / 搜索栏
12. Badge / Status Chip
13. 进度条 / 状态条
14. 通知项 / 活动日志项
15. 弹窗 / Popover / Dropdown

如确实需要新增组件，必须先映射到以上组件语义之一，再确定是否需要新命名。

### 19.13.1 当前可复用实现入口
当前 Desktop 前端已有以下共享实现，新增页面或重构时应优先复用，不要在业务页面里重复手写同类结构：

| 语义组件 | 实现入口 | 适用场景 |
| --- | --- | --- |
| 登录/权限门禁空状态 | `AuthGateCard` in `apps/desktop/src/ui/pageCommon.tsx` | 游客访问发布、管理、审核等需登录能力 |
| 通用空状态 | `SectionEmpty` in `apps/desktop/src/ui/pageCommon.tsx` | 空列表、无搜索结果、未选择对象、不可预览文件 |
| 状态/标签胶囊 | `TagPill` in `apps/desktop/src/ui/pageCommon.tsx` | neutral / success / warning / danger / info 五类语义状态 |
| 实体首字母标识 | `InitialBadge` in `apps/desktop/src/ui/pageCommon.tsx` | Skill、用户、部门、项目、审核对象的稳定 tone 标识 |
| 标准选择字段 | `SelectField` in `apps/desktop/src/ui/pageCommon.tsx` | 表单中带 label 的 select 控件 |
| 包文件预览 | `PackagePreviewPanel` in `apps/desktop/src/ui/pageCommon.tsx` | 发布包、审核包、Skill 包内容预览 |

抽取规则：
- 同一语义出现 3 次以上，才提升到共享组件或 token。
- 共享组件必须带清晰 props、语义化 className、键盘焦点样式和中文默认文案。
- 不抽取只有形似但意图不同的组件；例如管理危险区、包预览区和列表 Inspector 不应合并成过度通用容器。
- 新增 CSS token 必须表达语义，如 `--row-radius`、`--surface-panel-gradient`，不得为单个像素值创建临时 token。

## 19.14 新增页面的 AI 设计约束

### 19.14.1 AI 必须先回答的 6 个问题
在设计新页面前，AI 必须先明确：
1. 这个页面属于概览页、工作台页、市场页还是管理页？
2. 当前页面的主任务是什么？
3. 页面里的主操作是什么？
4. 页面里最重要的 1 个状态是什么？
5. 哪些信息必须同屏出现？
6. 空、错、加载、离线时页面如何收口？

### 19.14.2 AI 输出新页面时的固定结构
AI 在产出新页面设计时，必须按以下结构输出：
- 页面目标
- 页面模板类型
- 布局结构
- 组件清单
- 关键状态
- 动效说明
- 视觉 token 使用说明
- 与现有页面的继承关系

### 19.14.3 AI 不得做的事
- 不得另起一套配色
- 不得新增无来源的圆角、阴影和字体体系
- 不得把页面做成营销落地页
- 不得把所有内容包进多个嵌套卡片
- 不得为了“高级感”牺牲信息密度和主任务清晰度

---

## 19.15 新页面验收清单

新增页面在评审时，至少检查以下内容：

### 19.15.1 风格一致性
- 是否使用本规范中的主色、状态色和背景体系
- 是否使用本规范中的圆角梯度
- 是否与现有页面共享同一 Header / Sidebar 语法

### 19.15.2 结构一致性
- 是否已归类到四种标准模板之一
- 是否有明确主操作
- 是否存在结构重复或无意义装饰区

### 19.15.3 组件一致性
- 按钮、输入框、标签、卡片是否使用统一尺寸
- 图标尺寸与图标容器是否符合规则
- 状态标签是否采用统一色板

### 19.15.4 动效一致性
- 动效时长是否落在规范范围内
- 是否存在不必要的循环动效
- hover 是否控制在轻微浮起和提亮范围内

### 19.15.5 状态完整性
- 是否有空状态
- 是否有错误状态
- 是否有加载状态
- 是否有权限限制或离线态说明
