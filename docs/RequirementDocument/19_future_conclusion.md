# 15. 后续规划与需求结论

---

## 15.1 本期待后续细化项

以下内容已形成方向，但还需要进一步细化到交互稿或更细 PRD：

- 授权范围的具体选人 / 选部门控件交互
- 受限详情页是否支持"申请访问 / 申请使用"
- 项目继承公共 skill 的详细规则
- 工具自动检测支持哪些工具（注册表项 + 默认路径清单）
- 系统初审结果的具体风险标签与说明模板
- 首页推荐算法与综合排序权重
- 审核模板与拒绝原因模板
- manifest.json 的最终字段枚举值定义

---

## 15.2 二期迭代方向

| 方向 | 说明 |
|------|------|
| 审计日志 | 统一操作审计方案（谁、什么时间、做了什么） |
| 分类标签体系 | 预定义分类层级 + 用户标签规则 |
| 批量用户管理 | CSV 导入、批量角色调整、批量迁移部门 |
| 数据看板 | 全站 skill 统计、部门发布情况、审核通过率/拒绝率、平均审核时长 |
| 协作模型 | 多维护者、组织级发布、维护权转移 |
| 通知推送渠道 | 企业 IM 集成（飞书/企微/钉钉）、系统托盘通知 |
| 通知生命周期 | 保留期限、清理策略 |
| Skill 依赖解析 | 依赖自动安装、冲突检测（本期仅展示） |

---

## 15.3 需求结论

本系统本期应完成一个面向企业内部的 Skill Marketplace 基础闭环，核心包括：

- Skill 的统一定义与统一包规范
- 发布、审核、上架、下架、归档的完整流程
- 市场浏览、搜索、详情、排行榜能力
- 安装与启用分离的使用模型（Central Store + Tool Adapter 架构）
- 基于授权范围与公开级别的精细化治理
- 管理员审核与部门管理能力
- 自建账号体系与角色管理
- 工具、项目、通知等配套页面
- 离线场景基础支持
- 对权限收缩、版本覆盖、路径冲突、审核 SLA 的明确规则
- 为后续 MCP、插件、Agent 检索等能力预留扩展空间

该版本需求可作为后续：
- 产品原型设计
- 页面结构拆分
- 交互稿设计
- 前端任务拆解
- 后端接口模型讨论
- Skill 包格式标准化

的基础输入文档。
