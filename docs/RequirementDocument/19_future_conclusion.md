## 15.2 P2 迭代方向

P2 目标是补齐企业治理闭环，建议拆为"发布审核"和"管理治理"两个里程碑：

| 方向 | 说明 |
|------|------|
| 发布与版本更新 | 发布 Skill、重新提交、发布新版本、版本审核期间旧版本继续可用 |
| 审核工作台 | 待审核、审核中、已审核、锁单、退回、拒绝、审核历史 |
| 权限变更审核 | 权限扩大重新计算审核链路，权限收缩简化审核，审核期间原权限继续生效 |
| 部门/用户/Skill 管理 | 自建账号体系、角色分配、部门管理、Skill 下架/上架/归档 |
| 风险与模板 | 系统初审结果的具体风险标签与说明模板、审核模板与拒绝原因模板 |
| 分类标签体系 | 预定义分类层级 + 用户标签规则 |
| manifest 治理字段 | manifest.json 的完整字段枚举值定义 |
| 受限详情申请 | 评估是否支持"申请访问 / 申请使用" |

## 15.3 P3 迭代方向

P3 目标是生态接入、运营治理和规模化管理：

| 方向 | 说明 |
|------|------|
| 审计日志 | 统一操作审计方案（谁、什么时间、做了什么） |
| 批量用户管理 | CSV 导入、批量角色调整、批量迁移部门 |
| 数据看板 | 全站 skill 统计、部门发布情况、审核通过率/拒绝率、平均审核时长 |
| 协作模型 | 多维护者、组织级发布、维护权转移 |
| 通知推送渠道 | 企业 IM 集成（飞书/企微/钉钉）、系统托盘通知 |
| 通知生命周期 | 保留期限、清理策略 |
| Skill 依赖解析 | 依赖自动安装、冲突检测（P1/P2 仅展示） |
| 批量 Skill 目标管理 | 跨工具/项目的批量绑定、批量启用/停用、批量升级 |
| 公共 Skill 继承 | 项目继承公共 skill、禁用继承、组织策略下发 |
| MCP / 插件 / Agent 检索 | MCP 管理、开源 AI 工具插件管理、智能 Agent 搜索、RAG 检索增强 |

## 15.4 需求结论

本需求集应作为完整产品蓝图维护，但落地时按 P1 / P2 / P3 分阶段推进。

P1 Desktop 应完成面向企业用户的 Skill 使用闭环，核心包括：

- 登录、身份展示与服务端连接状态
- 市场浏览、搜索、详情与受限详情
- Skill 的安装、更新、卸载
- 已安装 Skill 的启用、停用和启用位置管理
- 工具与项目的基础路径配置
- 应用内通知、基础设置和离线本地使用
- P1 最小 Skill 包规范与 manifest 字段

P2 再补齐发布、审核、权限变更、部门/用户/Skill 管理等企业治理闭环。P3 再进入审计、看板、批量管理、多维护者、依赖解析、MCP/插件/Agent 检索等规模化能力。

该版本需求可作为后续：

- P1 Desktop PRD 与交互稿
- P2 发布审核与管理治理 PRD
- P3 生态扩展路线图
- 前后端接口模型讨论
- Skill 包格式标准化

的基础输入文档。
