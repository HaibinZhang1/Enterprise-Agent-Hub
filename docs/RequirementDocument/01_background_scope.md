# 1. 项目背景与目标

## 1.1 项目背景
企业内部会逐步沉淀大量面向不同 AI 工具、项目场景、部门业务的 skills。当前缺少一个统一的内部市场来完成以下事情：

- skill 的统一发布、审核、上架、下架、归档
- skill 的统一搜索、浏览、安装、启用、更新
- skill 的统一权限控制、部门范围管理、可见范围管理
- skill 的统一治理，包括审核、排行榜、版本管理、操作追踪

## 1.2 项目目标
搭建一个可在企业内网部署的 **Agent Skills 管理市场（Marketplace）**，让企业内部用户和管理员围绕 skill 完成以下闭环：

- 明确 skill 的定义、元信息规范与包结构规范
- 统一管理 skill 的发布、审核、展示、安装、启用与更新
- 支持按部门、权限、可见范围进行精细化控制
- 提供面向普通用户、发布者、审核员、管理员的统一操作入口
- 为后续扩展 MCP、插件管理、Agent 检索等能力预留空间

## 1.3 项目定位
本系统不是单纯的文件上传站，而是企业内部的 **skill 目录 + 审核中心 + 使用管理中心 + 治理平台**。

# 2. 范围与边界

## 2.1 本期范围
本期聚焦以下核心能力：

- **自建账号体系**（系统初始一个一级管理员，用户新增和角色由上一级管理员设置）
- 登录与身份区分
- 市场浏览与详情查看
- skill 发布与版本更新
- 审核流程与审核工作台
- 部门、用户、skill 管理
- 工具与项目维度的 skill 启用管理
- 通知（应用内消息中心）与首页待办
- 多语言基础支持
- Skill 包结构与元数据规范
- 离线场景基础支持

## 2.2 本期不做
以下内容本期仅预留，不进入正式交付范围：

- MCP 管理能力
- 开源 AI 工具插件管理
- 智能 agent 搜索
- RAG 检索增强
- 自动替用户完成卸载后的环境修复
- 完整系统安全扫描能力（本期只保留链路与状态，不做真实扫描实现）
- 审核通过后的高风险追溯复核机制
- 审核员离线 / 离职后的复杂接管机制
- 组织与人员变更后的自动迁移编排
- 审计日志（二期）
- 分类与标签体系定义（二期）
- 批量用户导入/管理（二期）
- 数据统计看板（二期）
- 多维护者/协作模型（二期）
- 通知推送渠道集成（二期）
- Skill 依赖自动解析（二期）
