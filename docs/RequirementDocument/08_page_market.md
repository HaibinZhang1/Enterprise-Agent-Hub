# 8.2 社区（原市场）

## 页面目标
社区承担企业内部 Skills、MCP 和插件的统一探索入口。已登录用户点击一级导航“社区”后进入社区首页；未登录用户点击时先弹出登录模态，登录成功后再进入社区首页。进入后用户可以直接搜索、查看榜单、进入 Skills 结果页，或进入发布工作区。

当前正式交付中，“市场”页面已经按新的 IA 收口为一级入口 **社区**。

> 说明：社区页属于当前版本 Desktop 核心范围。当前版本支持社区首页、搜索、排序、标签筛选、卡片网格、热榜侧栏，以及社区内作者工作区 `发布 / 我的`；MCP / 插件仅保留占位态。

## 页面结构
- 社区首页：独立完整页面，不显示左侧社区导航；包含顶部 Hero 搜索区、三个核心榜单和探索入口卡片
- 社区工作台：从首页点击探索或入口卡片后进入，左侧社区入口为 `首页 / Skills / MCP / 插件` + 作者分区 `发布 / 我的`
- Skills 结果页：位于社区工作台内，展示搜索与排序区、标签按钮行、Skill 卡片网格、右侧热榜侧栏

## 社区首页
社区首页是一级导航“社区”的默认落点，必须作为独立完整页面展示，不承载左侧导航栏。

### Hero 搜索区
- 标题：`EnterpriseAgentHub社区`
- 副标题：`发现、安装和管理企业内部可用的 Skills、MCP 和插件`
- 全局搜索框占位文案：`搜索 Skills、MCP、插件、作者、部门或标签`
- 主按钮：`探索`
- 次按钮：`发布`

行为规则：
- 未登录用户点击一级导航“社区”时，先弹出登录模态；登录成功后进入社区首页，取消登录则保持当前页。
- 直接点击 `探索` 或在搜索框提交后，进入带左侧导航栏的 `社区 > Skills` 结果页。
- 提交搜索时将关键词保留在现有社区 Skill 搜索框中；有关键词时默认使用相关度排序，无关键词时回到综合排序。
- 搜索提交会清理旧分类、标签等探索筛选，避免首页搜索被上一轮结果页筛选条件截断。
- `发布` 复用社区内作者工作区；未登录用户先触发登录门禁。

### 三个核心榜单
首页首屏中间同时展示三个并列榜单卡片：
- 热榜
- Star 榜
- 下载榜

每个榜单展示 Top 10，并复用当前社区榜单行设计。服务端榜单优先；离线或未获取服务端榜单时，Star 榜与下载榜使用本地缓存数据派生，热榜无近期数据时展示空态。

### 探索入口区
榜单下方展示三个入口卡片：
- 技能探索：进入带左侧导航栏的当前 `社区 > Skills` 卡片结果页
- MCP 探索：进入带左侧导航栏的 MCP 预留页
- 插件探索：进入带左侧导航栏的插件预留页

## 搜索能力
支持按以下字段搜索：
- skill 名称
- 描述
- 标签
- 作者
- 部门
- skillID

## 筛选能力
支持按以下维度筛选：
- 分类：固定中文短分类单选，包含 `开发 / 测试 / 文档 / 设计 / 运维 / 安全 / 集成 / 自动化 / 数据 / 知识 / 其他`
- 标签：固定中文短标签多选，默认 OR 逻辑，命中任一标签即可
- 排序
- 其他高阶筛选维度仍由服务端字段逐步增强

当前正式页面优先交付：
- 搜索词
- 分类行
- 标签多选行
- 排序

## Skill 卡片

社区当前采用卡片网格，而不是单列大摘要列表。

每张卡片展示：
- 首字母图标
- 显示名称
- 简短描述
- 分类
- 最多 3 个标签，超出显示 `+N`
- 当前状态
- 作者
- 作者部门
- 当前版本
- Star 数
- 下载量

### 卡片操作
- 主操作：点击卡片进入详情覆盖层
- 次操作：收藏 / 取消收藏
- 安装 / 更新 / 启用范围等动作当前集中在详情覆盖层中处理
- 安装下载凭证发放并完成本地写入后，前端需刷新该 Skill 详情并即时更新下载量、热榜和详情覆盖层中的统计展示

## Skill 详情

当前正式交付的 Skill 详情优先通过覆盖层展示，而非独立详情页。

完整详情展示：

详情覆盖层采用单页阅读结构：顶部为 Skill 基础信息，操作按钮固定在覆盖层右上角；中部为概览、兼容与已启用位置；README 之前展示版本历史，底部展示 `README.md` 的完整 Markdown 内容。

### 基础信息
- skillID
- 显示名称
- 描述
- 作者
- 所属部门
- 当前版本
- 本地版本
- 更新时间
- 公开级别

### 使用与兼容信息
- 适用工具
- 支持环境说明
- 分类与标签
- 已启用位置

### 内容信息
- 版本历史：按发布时间倒序展示版本号、发布时间、风险等级、变更说明，并标记当前版本
- `README.md`：作为详情底部主阅读区，展示 Skill 包中 README.md 的完整 Markdown 内容；缺失时显示空态

### 审核与安全信息
- 风险等级
- 审核意见摘要
- 风险说明

### 操作区
操作按钮位于详情覆盖层右上角，关闭按钮保留在最右侧。

- 安装
- 更新
- 收藏 / 取消收藏
- 启用范围
- 查看已启用位置

## 受限详情
当用户属于“摘要公开但无详情权限”时，仅展示：
- skill 名称
- 发布人
- 部门
- Star 数
- 下载量
- 分类 / 图标（如有）
- “该 skill 暂未向你开放详情”提示

## 热榜侧栏

社区右侧保留独立热榜侧栏，不再回到主页。

当前版本默认展示简单热榜能力：
- 热榜
- 下载榜
- Star 榜
- 默认展示热榜
- 用户通过页签手动切换热榜、Star 榜与下载榜；榜单不自动轮播
- 榜单行点击后进入 Skill 详情覆盖层

榜单行字段：
- 首字母图标
- Skill 名称
- 单个分类标签
- 发布者名称
- 发布者部门
- Star
- 下载量

展示规则：
- 单次展示 10 条
- 三个榜单不同时上下排列，只展示当前手动选中的榜单
- 热榜按近 7 天趋势热度排序：`hotScore = (recentDownloadCount * 6 + recentStarCount * 3 + updateBoost) * riskMultiplier`
- `recentDownloadCount` 为近 7 天安装下载凭证按用户去重数；`recentStarCount` 为近 7 天新增且当前仍有效的 Star 数；当前版本更新时间在近 7 天内时 `updateBoost = 2`，否则为 `0`
- 风险倍率：低风险和未知风险为 `1.0`，中风险为 `0.8`，高风险为 `0.5`
- 热榜只展示 `hotScore > 0` 的 Skill；并列时按近 7 天安装下载用户数、近 7 天新增有效 Star、累计下载量、累计 Star、最近更新时间、名称排序
- 下载榜按安装下载用户数降序排列，Star 作为并列时的次级排序；Star 榜按 Star 数降序排列，下载量作为并列时的次级排序；仍并列时按名称排序
- 下载量不统计更新、文件预览、审核包下载或同一用户的重复安装下载凭证

## 发布入口

发布中心不再作为一级导航，当前实现收口在社区内部作者分区。

当前社区通过左侧作者入口提供：
- `发布`
- `我的`

行为规则：
- 已登录：进入社区内对应作者工作区
- 未登录：先打开登录模态，再进入对应作者工作区
