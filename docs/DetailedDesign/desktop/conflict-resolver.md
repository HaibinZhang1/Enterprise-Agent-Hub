# conflict-resolver 模块设计

## 模块目标
处理工具级/项目级安装冲突、路径冲突、版本覆盖冲突与用户决议持久化。

## 主要职责
- 冲突检测
- 冲突类型分类
- 给出决策建议（覆盖/并存/取消）
- 存储用户决议并复用

## 本地状态
- `desktop_conflicts`
- `desktop_conflict_resolutions`

## 交互
- IPC：本地路径冲突、文件占用、目标已存在检查
- 前端：冲突弹窗与决议确认

## 失败/恢复
- 决议前默认不执行破坏性覆盖
- 冲突解决失败后返回可重试态

## 风险/权衡
- 自动裁决会减少操作，但可能误伤本地自定义文件；一期以显式用户确认为主
