# tool-scanner 模块设计

## 模块目标
识别本机已安装 AI 工具、探测默认路径与用户自定义路径，为后续 Skill 分发提供 Tool Adapter 上下文。

## 主要职责
- 扫描注册表、默认目录、marker files
- 识别工具版本、安装路径、可写性
- 维护工具发现结果缓存
- 将异常路径与不可写状态反馈给 UI

## 本地状态
- `desktop_tools`
- `desktop_tool_paths`
- `desktop_tool_scan_runs`

## 交互
- IPC：供前端工具页触发重新扫描
- HTTP：通常无；仅消费服务端工具元配置

## 失败/恢复
- 扫描失败不阻塞其他工具展示
- 允许用户手动修正路径并重新验证

## 风险/权衡
- 默认路径规则要可配置，不把工具路径硬编码死
