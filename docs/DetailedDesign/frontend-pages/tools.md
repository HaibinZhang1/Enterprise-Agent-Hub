# tools 页面设计

## 页面目标
展示已识别工具、默认路径结果、自定义路径配置与当前启用状态。

## 页面区域
- 工具列表
- 扫描结果/健康状态
- 路径配置表单
- 刷新检测按钮
- 单工具、单 Skill 的启用/停用预览确认区

> V1 边界：工具页不得提供批量绑定、批量启用/停用或批量升级入口；所有会影响本地 `skills/` 目录的动作都应限定为一次一个明确工具目标，并通过预览确认执行。可在布局上为后续批量工作流预留空间，但不得展示为当前可用能力。

## 关键数据与动作
- IPC `scanTools`
- IPC `validateToolPath`
- 本地 SQLite 查询工具缓存

## 状态设计
- 未检测到工具时提供引导
- 路径无写权限时标红并提示修复

## 权限要点
- 主要是本机权限校验，无服务端角色限制
